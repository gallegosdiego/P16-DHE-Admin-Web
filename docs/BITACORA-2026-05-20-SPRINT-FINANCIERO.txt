ENDPOINTS API NUEVOS (8)
================================================================================

  GET /api/financial/kpis
    → KPIs financieros consolidados (DSO, tasa COD, márgenes, etc)

  GET /api/financial/aging-report
    → Reporte de antigüedad de cartera por cliente

  GET /api/financial/cash-flow?weeks=13
    → Proyección de flujo de caja semanal

  GET /api/financial/profitability/by-zone
    → Rentabilidad por zona de entrega

  GET /api/financial/profitability/by-driver
    → Rentabilidad por conductor

  GET /api/financial/profitability/by-client
    → Rentabilidad por cliente

  GET /api/financial/driver-settlement/{driver}?from=&to=
    → Liquidación detallada de un conductor por período

  GET /api/financial/alerts
    → Alertas financieras activas


MÓDULO FINANCIERO — 7 TABS
================================================================================

  Tab 1: Dashboard Financiero
    - 6 KPI cards (ingreso, costos, utilidad, DSO, tasa COD, margen/envío)
    - Alertas financieras activas (badges de color)
    - Barra progreso COD del día
    - Mini P&L del mes

  Tab 2: Estado de Resultados (P&L)
    - Selector de período (desde/hasta)
    - Tabla contable profesional (ingresos/costos/utilidad)
    - Comparativo visual
    - Exportar CSV

  Tab 3: Cartera (Aging Report)
    - 6 buckets de antigüedad con KPI cards
    - Tabla detallada por cliente con semáforo visual
    - Filtros: todos, vencidos, >90 días
    - Acción WhatsApp cobro
    - Exportar CSV

  Tab 4: COD y Conciliación
    - Resumen diario por conductor
    - Crear conciliación manual
    - Historial de conciliaciones
    - Alertas de COD no depositado

  Tab 5: Conductores (Liquidaciones)
    - Tablero de recaudo (COD pendiente/cobrado/por pagar por conductor)
    - Tabla de rentabilidad por conductor (envíos, ingreso, pagado, margen)
    - Liquidación individual flexible con selector conductor + período
    - Detalle de entregas + resumen COD
    - Exportar liquidación CSV
    - Acciones batch: recaudar, liquidar, pagar

  Tab 6: Gastos y Nómina
    - Panel split: gastos fijos | nómina
    - CRUD gastos con historial expandible
    - Registrar pagos de nómina con historial
    - Totales mensuales

  Tab 7: Flujo de Caja
    - Proyección 13 semanas en tabla horizontal
    - Filas: saldo inicial, entradas (clientes, COD), salidas (conductores,
      gastos, nómina), flujo neto, saldo final
    - Código de color verde/rojo
    - Exportar CSV


CORRECCIONES APLICADAS
================================================================================

  1. Dark Mode — Tailwind v4 no respetaba toggle manual
     Causa: Tailwind v4 usa @media prefers-color-scheme por defecto
     Fix: Se agregó @custom-variant dark en globals.css
          + ThemeProvider ahora pone clase 'dark' y data-theme juntos

  2. Logo sidebar — Muy pequeño y recortado
     Fix: Logo horizontal grande (h-10), sin cuadro contenedor,
          texto "Panel Admin" subtítulo centrado

  3. Login — Sin imagen de marca, solo dark
     Fix: Imagen de fondo real de la marca + card glassmorphism

INSTRUCCIONES PARA CONTINUAR EN CASA
================================================================================

  1. Clonar/actualizar:
     git pull origin main

  2. Backend:
     cd api
     php artisan serve --port=8000

  3. Frontend:
     cd frontend
     npm install    (si hay dependencias nuevas)
     npm run dev

  4. Acceder:
     http://localhost:3000/login
     admin@danheiexpress.com / (contraseña habitual)

  5. Verificar fondo login:
     - Abrir http://localhost:3000/login-bg.png directo
     - Si carga: hacer Ctrl+Shift+R en /login
     - Si no carga: verificar que login-bg.png esté en frontend/public/
